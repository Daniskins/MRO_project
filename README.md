# MRO Service — Backend Platform (FastAPI)

Backend-платформа для учета и управления объектами технического обслуживания (MRO): воздушные суда, агрегаты, ресурсы и наработка.

Проект разрабатывается как **инженерный pet-проект и портфолио**, с фокусом на промышленный подход, чистую архитектуру и масштабируемость, а не на «учебный CRUD».

---

## О проекте

В авиационной и промышленной среде до сих пор широко используются Excel-таблицы, ручной учет наработки и разрозненные системы.  
Цель проекта — **спроектировать backend-основу MRO/CMMS-системы**, которая:

- отражает реальные бизнес-сущности
- легко расширяется
- не привязана к конкретному фронтенду
- готова к интеграции аналитики и ML-модулей

---

## Технологический стек

- **Python 3.11+**
- **FastAPI** — высокопроизводительный REST API
- **SQLAlchemy 2.0 (async)** — современная ORM
- **Alembic** — контроль версий схемы БД
- **Pydantic v2** — строгая валидация данных
- **PostgreSQL** 
- **Uvicorn**

---

## Архитектурные принципы

Проект строится с опорой на практики, используемые в реальных backend-системах:

- четкое разделение слоев:
  - API (FastAPI routers)
  - бизнес-логика (CRUD / сервисы)
  - модели данных
  - инфраструктура
- минимизация связности между модулями
- предсказуемые зависимости
- готовность к росту проекта без переписывания ядра

Архитектура изначально проектировалась так, чтобы в дальнейшем можно было добавить:
- роли и авторизацию
- аудит операций
- WebSocket-уведомления
- аналитические и ML-модули
- возможно телеграмм-бот для уведомлений

---

## Структура проекта

- app/
  - core/ — Конфигурация и настройки
  - crud/ — Бизнес-логика и операции CRUD
  - database/ — Модели БД и сессии
  - routers/ — FastAPI роутеры
  - schemas/ — Pydantic модели
  - static/ — Статические файлы (стили, скрипты)
  - templates/ — Шаблоны Jinja2
  - utils/ — Дополнительные сервисы и утилиты
  - main.py — Точка входа приложения


---

## Текущий статус

Проект находится в активной разработке. На данный момент реализованы:
- базовая структура проекта
- модели данных для воздушных судов
- CRUD-операции для основных сущностей
- миграции Alembic
- базовые роутеры FastAPI

В планах:
- расширение моделей данных (двигатели, агрегаты, ресурсы)
- добавление ролей и авторизации
- Docker-контейнеризация
- написание тестов
